\subsection*{А.1. Промпт для машинного перевода текстовых полей}

Для перевода текстовых полей RAGBench-RU использовался единый минимальный
промпт к модели Qwen2.5-72B-Instruct. Модель получала отдельный текстовый
фрагмент и возвращала только его русский перевод без дополнительных рассуждений
или комментариев.

\begin{verbatim}
SYSTEM:
Ты — профессиональный переводчик с английского на русский.
Переводи точно и понятно. 
Рассуждения и дополнительные пояснения не нужны; верни только перевод.

USER:
{text}
\end{verbatim}

Сохранение структуры примеров не задавалось отдельным JSON-промптом, а
обеспечивалось на уровне конвейера: текстовые поля переводились отдельно,
ключи предложений сохранялись из исходного корпуса, после чего выполнялись
структурные проверки числа предложений и наличия ключей разметки.

\subsection*{А.2. Промпт LLM-оценщика без дообучения (Qwen2.5-72B)}

Промпт, использованный в сравнительном эксперименте оценки без дообучения на базе
Qwen2.5-72B (см. раздел~\ref{sec:classifier_vs_llm}), соответствует англоязычной
схеме промпта из оригинальной работы RAGBench. Оценщик не используется для
построения разметки корпуса RAGBench-RU.

Модели передавались документы с ключами предложений, вопрос и ответ. На выходе
запрашивался JSON-объект с полями
\texttt{relevance\_explanation},
\texttt{all\_relevant\_sentence\_keys},
\texttt{overall\_supported\_explanation},
\texttt{overall\_supported},
\texttt{sentence\_support\_information} и
\texttt{all\_utilized\_sentence\_keys}.

Поле \texttt{all\_relevant\_sentence\_keys} содержит ключи предложений документов,
релевантных вопросу. Поле \texttt{all\_utilized\_sentence\_keys} содержит ключи
предложений документов, использованных при построении ответа. Поле
\texttt{overall\_supported} отражает, поддерживается ли ответ в целом документами,
а \texttt{sentence\_support\_information} содержит информацию о поддержке отдельных
предложений ответа.

В последующем количественном анализе использовались только поля
\texttt{all\_relevant\_sentence\_keys} и
\texttt{all\_utilized\_sentence\_keys}; поле
\texttt{overall\_supported} и подробная информация
\texttt{sentence\_support\_information} не входили в метрики
таблицы~\ref{tab:llm_errors}. Полный текст промпта приведён в ноутбуке
\texttt{test\_llm\_razm.ipynb}.

(совпадение, пропуски, лишние ключи) на тестовых поддатасетах исходного
англоязычного RAGBench. \emph{overlap} - доля ключей, совпавших с эталоном;
\emph{miss} - среднее число пропущенных ключей на пример; \emph{extra} -
среднее число лишних. Метрики получены при запуске LLM-оценщика на
англоязычных примерах с оригинальным англоязычным промптом из статьи

опорным ориентиром при сравнении с обученным классификатором, оценённым на
